# Idea — Cross-app typed-card interchange standard & registry

> Layer-A Idea node (durable · brand-free). Model: [`STRATEGIC-LANDSCAPE-MODEL.md`](../../STRATEGIC-LANDSCAPE-MODEL.md). **Bootstrap seed.**

- **Topic(s):** [structured-documents-and-cards](../TOPICS/structured-documents-and-cards.md).
- **One-line:** An ecosystem-wide vocabulary of **typed content "cards"** — namespaced `Producer::type` interchange/registry types that any app can post and any app can render identically (the interoperability spine that makes "an ecosystem, not an app" self-evident).
- **Axes (Idea-level):** Conviction = **active** as a live cross-venture concept; the **namespacing/registry seam is undecided** · Horizon = rendering-layer slice **now** (a fully-built app exists), the namespaced standard + registry **~3yr / far** · Provenance = the `divia_cards` rendering app (Nov 2025, the name-source) · Leverage = Svelte 5 → framework-agnostic Web Components + Shadow DOM + CSS-Custom-Property theming ("one card definition renders identically everywhere"); the Activity-Log NL-capture card as an ecosystem primitive.
- **Idea↔Idea edges:** `enables` → [structured-knowledge document format](structured-knowledge-document-format.md) (the `.dvai` document is an outline *of* these cards) · `enables` → [cross-app integration standard](cross-app-integration-standard.md) (cards are the public payload/wire contract over the network) · sibling-of [graph-DB entity-knowledge engine](graph-db-entity-knowledge-engine.md).
- **Channel (Idea→Venture, time-bounded):** [DiviaCards](../PRODUCTS/DiviaCards/divia_cards.md) (its own open-standard umbrella under Divia.AI, Inc.; status 🟢 active) — owned/defined by [Divia.AI Professional](../PRODUCTS/DiviaAI/divia_ai-professional.md); headline future consumer = PatternicityNews/PatternicitySocial.
- **Realized by (Build Lines, Layer B):** the `divia_cards` rendering-layer Build Line (Flask + SvelteKit → `divia-cards` MIT Web-Components npm package; 3 flat card types `nlp_input`/`outline`/`event`, **no namespacing/registry yet**). The namespaced standard is a forward/open second Build Line, not yet built. See [engineering brief](../../SOFTWARE_DEV/divia_cards.md).
