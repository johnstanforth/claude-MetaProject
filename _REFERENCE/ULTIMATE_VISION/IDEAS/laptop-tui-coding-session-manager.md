# Idea — Laptop TUI multi-tool coding-session manager

> Layer-A Idea node (durable · brand-free). Model: [`STRATEGIC-LANDSCAPE-MODEL.md`](../../STRATEGIC-LANDSCAPE-MODEL.md). **Bootstrap seed.**

- **Topic(s):** [developer-tooling-ai-coding](../TOPICS/developer-tooling-ai-coding.md).
- **One-line:** A Midnight-Commander-style terminal/TUI that **wraps the CLI coding assistants** developers use daily (Claude Code, Codex, Gemini…) and manages many instances at once in organized workspaces — the local cockpit and the laptop-side bridge to the platform.
- **Axes (Idea-level):** Conviction = **active** (shipping; most product-complete app in the whole portfolio) · Horizon = **now / v1** (Phases 0–7 complete, Phase 8 in progress) · Provenance = John's "power-user, see-under-the-hood" bridge between CLI tools and the platform · Leverage = Python Prompt Toolkit (chosen to map cleanly to a future Rust/ratatui migration); SQLite WAL; the multi-tool wrapping pattern.
- **Idea↔Idea edges:** sibling-of [lossless AI-coding-session archive](lossless-ai-coding-session-archive.md) (bundled in aixocode but a distinct customer-value — orchestration vs. preservation) · hosts [cross-vendor agent collaboration bus](cross-vendor-agent-collaboration-bus.md) (CollabPairs run here) · `enables` [context-preserving search](context-preserving-search.md) (a candidate home: a "scoped project search" tool) · sibling-of [cross-platform AI-coding desktop](cross-platform-ai-coding-desktop.md) (a **different** Idea — offline desktop, not a laptop TUI).
- **Channel (Idea→Venture, time-bounded):** [AIXO.Dev Platform / ExoDev](../PRODUCTS/AIXO.Dev/aixodev-platform.md) (active) — realized by `aixocode`. **License: MIT** (the only MIT product among proprietary AIXO.Dev siblings — a deliberate open-core top-of-funnel wedge).
- **Realized by (Build Lines, Layer B):** `aixocode` — MC-style four-zone TUI (sidebar tree, tab bar, F-key bar, 5 themes, fuzzy command palette, snapshots), three operating modes (default **TUI**, `--tmux-session` via libtmux, `--direct` PTY passthrough), CLI suite (`scan`/`import`/`archive`/`stats`/`sync`/`hook-event`/`config`), plus the `aixotest` visual-testing tool. See the [engineering brief](../../SOFTWARE_DEV/aixodev-platform.md).
