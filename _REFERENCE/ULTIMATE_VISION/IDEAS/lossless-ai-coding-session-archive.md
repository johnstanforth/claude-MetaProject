# Idea — Lossless AI-coding-session capture & archive

> Layer-A Idea node (durable · brand-free). Model: [`STRATEGIC-LANDSCAPE-MODEL.md`](../../STRATEGIC-LANDSCAPE-MODEL.md). **Bootstrap seed.**

- **Topic(s):** [developer-tooling-ai-coding](../TOPICS/developer-tooling-ai-coding.md).
- **One-line:** Capture every AI-coding-tool session **byte-for-byte** (`raw_data`/`raw_message` preserved, original JSONL intact) into a per-tool archive that is never deleted, truncated, or schema-changed — an irreplaceable historical record that becomes a compounding moat.
- **Axes (Idea-level):** Conviction = **active** (shipping; the highest-priority architectural constraint in aixocode) · Horizon = **now / v1** (Claude Code archival works today; other-tool parsers planned) · Provenance = John's preservation discipline; framed as a moat in the Phase-D research · Leverage = SQLite (WAL) archive + the cross-tool parser roadmap; the byte-for-byte guarantee.
- **Idea↔Idea edges:** `enables` → [software-dev knowledgebase / Ontology graph](software-dev-knowledgebase-ontology-graph.md) (the archive is the connective tissue / mineable corpus) · `enables` → [context-preserving search](context-preserving-search.md) (a mineable corpus to search) · sibling-of [laptop TUI session manager](laptop-tui-coding-session-manager.md) (bundled in aixocode but a distinct customer-value).
- **Channel (Idea→Venture, time-bounded):** [AIXO.Dev Platform / ExoDev](../PRODUCTS/AIXO.Dev/aixodev-platform.md) (active) — realized laptop-side by `aixocode`, ingested server-side by `aixodev-web`.
- **Realized by (Build Lines, Layer B):** `aixocode` (per-tool `*_sessions.db`, scan/import, 22 hook event types) **and** `aixodev-web` (the session-ingest API `POST /api/v1/sessions/ingest`, raw-JSONB preservation, transcript viewer) — see the [engineering brief](../../SOFTWARE_DEV/aixodev-platform.md).
- **Notes:** A `[DEALBREAKER-HOOK]` per the engineering brief: the byte-for-byte guarantee is the irreversible substrate competitors can't retrofit ("years of real human+AI development captured losslessly"). The `aixodev-openhands` research says AIXO must **decline** OpenHands' file-based per-conversation persistence precisely because it conflicts with this guarantee. Long parser roadmap (Codex, Gemini, aider, Cline, Roo, Goose, OpenCode, IDE-conversation parsers). The **AIXO MCP server** (exposing session data/analytics as MCP tools) is an interface over this archive, not a separate Idea.
