# Idea — Cloud orchestration platform (CloudXMT → SensoryMQ.Cloud)

> Layer-A Idea node (durable · brand-free). Model: [`STRATEGIC-LANDSCAPE-MODEL.md`](../../STRATEGIC-LANDSCAPE-MODEL.md). **From John's first-hand history (2026-06-22).**

- **Topic(s):** [Cloud Orchestration & DevOps](../TOPICS/cloud-orchestration-devops.md) · [IoT & Connected Devices](../TOPICS/iot-connected-devices.md).
- **One-line:** A hosted cloud platform whose **central orchestration layer auto-generates and prepackages Docker containers from web-IDE configuration, then pushes and auto-scales them in production** — originally **CloudXMT** (the GridTransmit-era hosted IoT platform; a Kubernetes-*precursor*) → reimagined as **SensoryMQ.Cloud**.
- **Axes (Idea-level):** Conviction = **validated-but-unlaunched / reboot-candidate** · Horizon = **past** (CloudXMT) + **future** (SensoryMQ.Cloud) · Provenance = CloudXMT announced ~2014/2015 (on stage in the GridTransmit award speech); broadly **validated by the 2018 OpenX project** · Leverage = one orchestration engine reused across many use-cases (IoT and beyond).
- **Idea↔Idea edges:** `predecessor` CloudXMT → SensoryMQ.Cloud · `related-to` [iot-connected-device-platform](iot-connected-device-platform.md) (the IoT productization) · `leverages` [RosettaMQ](rosettamq.md) (SensoryMQ.Cloud's proprietary IoT calls layer onto open-source RosettaMQ).
- **Channel (Idea→Venture, time-bounded):** **CloudXMT** (a GridTransmit product-line, ~2014/2015) → **SensoryMQ.Cloud, LLC** (idea, ~2017/2018–).
- **Notes:** Headline capability — a web-IDE call like `new_device = TMobile_M2M_Provider.provision_new_device()` automating what was a manual **6-week-plus carrier contract negotiation** — extending IoT to an AWS-style cloud model **before AWS had IoT features**. **Never fully launched:** Google's **Kubernetes** announcement (forwarded by John's long-time collaborator **David Lang**, then at Google HQ Mountain View) captured the orchestration mindshare, making a proprietary alternative hard to pitch. **OpenX validation (late-summer 2018):** reconfigured / upgraded / indexed **53,000+ server instances** across geo-distributed datacenters for OpenX's (Pasadena ad-tech) co-lo→Google-Cloud migration — quoted **<$40K / 6 weeks** vs. outsourcers' **$75K+ / 3 months**, delivered via **2 weeks on-site modeling + ~2 days of orchestration execution**. Per John (2026-06-22).
