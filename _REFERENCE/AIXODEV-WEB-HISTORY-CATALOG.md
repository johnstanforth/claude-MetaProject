## 0. Orientation — What `aixodev-web` Is, and Its Phase Skeleton

`aixodev-web` is the **server-side web application** of the AIXO.Dev Platform: a Flask + PostgreSQL + HTMX/Tailwind dev-project-management platform (issues, wiki, GitHub, analytics, notifications) that also ingests and renders AI coding-session transcripts (Claude Code JSONL, Claude.ai exports), and receives real-time hook events from the companion `aixocode` CLI client. Tech stack as built: Python 3.11 / Flask 3.x / SQLAlchemy 2.0 / PostgreSQL 16 / Redis 7 / Flask-SocketIO (eventlet→gevent→threading over time) / HTMX 2.0 / Tailwind 3.4 / Chart.js 4 / highlight.js / Dramatiq+Periodiq workers / `uv`+`pyproject.toml`+`pydantic-settings`+Ruff/mypy/pytest. ## PHASE 5 — Entity Model & Platform Vision (4 sprints shipped, IN PROGRESS)

> A research-heavy phase that **specified** (did not implement) an expansion from 25→~75 tables across 12 entity families, and **built** supporting infrastructure (debugging, ops, staging, session-ingest API) operationalizing the `aixodev-web` (server) ↔ `aixocode` (CLI client) split. (Source: `phase_05--entity_model_and_platform_vision/`)

**`DECISIONS.md` is effectively empty** (placeholder only); the real architectural decisions live in §16 of the entity-model spec:
- **Hierarchy strategy = ltree** (PostgreSQL `ltree` for deep/queried hierarchies — projects/topics/dev_tasks/wiki; adjacency list for shallow ones; both representations coexist).
- **Agent vs User = SEPARATE entity** (agents have model/persona/routing, token budgets, distinct audit/billing/retire; unified in UI via `DevTeamMembership.member_type='person'|'agent'`).
- **Issue vs DevTask = SEPARATE, linkable** (Issues = external-facing requests; DevTasks = execution work assignable to humans AND agents; one Issue → 0..N DevTasks).
- **Polymorphic pattern** = string `entity_type` discriminator + nullable per-type FK.
- **Naming conventions** = plural snake_case tables, singular PascalCase models, VARCHAR(20) status strings (not enums).
- **Client-server separation** (recorded in `sp_04` + `_research/ARCHITECTURE_ANALYSIS--aixocode-vs-aixodev-web.md`): `aixodev-web` = authoritative Flask/PG server; `aixocode` = per-developer CLI doing hook capture + JSONL parse + SQLite archival + queued sync.
